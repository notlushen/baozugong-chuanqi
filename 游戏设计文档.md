# 《包租公传奇》游戏设计文档

> **引擎**: Godot 4.6 | **类型**: 放置类 / 轻度策略 | **视觉**: 2D 卡通/像素 | **平台**: PC

---

## 一、游戏概述

### 1.1 核心概念
玩家扮演一名初入行的房东，通过收租积累财富，购买更多房产，解锁更高级的物业类型。被动收租 + 主动小游戏双轨驱动，让玩家在挂机与操作间自由切换。

### 1.2 核心循环
```
收租赚钱 → 购买/升级房产 → 解锁新物业类型 → 声望重置 → 更高倍率
    ↑                                              ↓
    └────────── 小游戏赚取额外资金 ─────────────────┘
```

### 1.3 游戏目标
- **短期**: 赚取足够资金购买下一套房产
- **中期**: 解锁所有物业类型，完成声望重置
- **长期**: 达到天文数字财富，解锁隐藏成就

## 三、房产系统

### 3.1 物业类型与解锁条件

| 序号 | 物业类型 | 图标 | 基础价格 | 基础租金/秒 | 解锁条件 |
|------|----------|------|----------|-------------|----------|
| 1 | 🏠 单间公寓 | 🟢 | 100 | 5 | 初始解锁 |
| 2 | 🏢 两居室 | 🔵 | 1,100 | 28 | 累计赚取 500 |
| 3 | 🏘️ 联排别墅 | 🟡 | 12,000 | 140 | 累计赚取 5,000 |
| 4 | 🏬 商业商铺 | 🟠 | 130,000 | 780 | 累计赚取 50K |
| 5 | 🏗️ 写字楼 | 🔴 | 1,400,000 | 4,200 | 累计赚取 500K |
| 6 | 🏰 豪华酒店 | 🟣 | 20,000,000 | 26,000 | 累计赚取 5M |
| 7 | 🌆 摩天大楼 | ⚪ | 330,000,000 | 150,000 | 第一次声望重置后 |

### 3.2 核心数值公式

#### 房产价格增长
```
购买第 n 套同类房产的价格 = 基础价格 × 1.15^n
```
- **增长率 1.15**: 经典放置游戏参数，保证约 60-120 秒可购买下一套
- 每购买一套，下一套价格上涨 15%

#### 租金收入计算
```
单套房产每秒租金 = 基础租金 × (1 + 0.08)^(等级-1) × 全局倍率 × 声望倍率
```
- **等级增长 8%**: 升级房产提升租金效率
- **全局倍率**: 来自升级/成就的临时或永久加成
- **声望倍率**: 声望重置后获得的永久加成

#### 总被动收入
```
每秒总收入(IPS) = Σ(每套房产每秒租金)
```

#### 投资回报周期 (ROI)
```
ROI(秒) = 房产价格 / 每秒租金
```
- 目标 ROI 范围: **60-180 秒**
- 新玩家约 60 秒回本，后期约 180 秒（保持挑战性）

### 3.3 房产三维成长系统

房产系统采用**三个维度**的独立成长路径，防止玩家死磕单一房产：

| 操作 | 战略定位 | 效果 | 限制 |
|------|----------|------|------|
| **购买新房产** | **解锁机制/全局被动** | 解锁该房产类型，每种新房产有独特被动效果 | 价格随累计购买数指数增长 |
| **购买已有房产** | **提升升级上限** | 增加该类型房产的**数量** | 每套租金 × 套数 |
| **升级已有房产** | **提升单套效率** | 提升该类型房产的**等级** | **等级上限 = 拥有数量** |

#### 升级上限机制
```
最高等级 = 该房产拥有数量
```
- 拥有 1 套 → 最高 Lv.1
- 拥有 3 套 → 最高 Lv.3
- **强制玩家横向扩展**，防止死磕单一房产

#### 房产卡片显示格式
```
🏠 单间公寓
拥有: 2 (Lv.1/2)          ← 当前等级/最高等级
每套: +5.4/秒 | 总计: +10.8/秒
[购买] 💰 115    [升级] 💰 50
```

#### 升级费用表
| 等级 | 租金倍率 | 升级费用 |
|------|----------|----------|
| Lv.0 → Lv.1 | 1.08x | 基础价格 × 0.5 |
| Lv.1 → Lv.2 | 1.17x | 基础价格 × 1.2 |
| Lv.2 → Lv.3 | 1.26x | 基础价格 × 2.5 |
| Lv.3 → Lv.4 | 1.36x | 基础价格 × 5.0 |
| Lv.4 → Lv.5 | 1.47x | 基础价格 × 8.0 |
| 后续 | 每级 +8% | 指数增长 |

### 3.4 收租机制（待领取池）

租金收入不直接进入金币，而是累积到**待领取租金池**，玩家点击底部"收租"按钮手动领取。

```
被动收入 → 待领取租金池 → [点击收租] → 金币 + 飞币动画
```

**设计意图**：
- 增加玩家的**参与感**和**爽感**
- 收租时播放金币飞出动画和数字脉冲效果
- 收租按钮显示可领取金额：`🏠 收租 (💰 1.5K)`

### 3.5 开始界面与存档管理

游戏启动时先进入**开始界面**（`start_screen.tscn`），而非直接进入游戏。

| 按钮 | 功能 |
|------|------|
| **📂 继续游戏** | 读取存档进入主游戏（无存档时禁用） |
| **🎮 新的游戏** | 清除旧存档，以 500 金币初始状态开始 |
| **🗑️ 删除存档** | 删除存档文件（仅在有存档时显示） |

**存档信息展示**（有存档时显示）：
- 💰 金币数量
- 🏠 房产套数
- ⭐ 声望等级

## 四、小游戏系统

### 4.1 小游戏分类

小游戏分为 **随机事件** 和 **固定活动** 两类：

#### A. 随机事件（被动触发）

| 事件名称 | 触发条件 | 类型 | 奖励范围 | 描述 |
|----------|----------|------|----------|------|
| 🚪 催租 | 每 2-5 分钟 | 限时点击 | 10-30 秒租金 | 点击催促拖欠房租的租客 |
| 🔧 维修 | 每 3-8 分钟 | 快速反应 | 5-15 秒租金 | 在正确时机点击修复设施 |
| 🎉 租客庆祝 | 拥有 5+ 房产时随机 | 记忆匹配 | 20-50 秒租金 | 记住并重复租客的动作序列 |
| 💼 大单生意 | 声望 ≥ 1 时随机 | 选择判断 | 30-100 秒租金 | 从多个合同中选择最优方案 |

## 五、声望系统（Prestige）

### 5.1 声望重置机制

当玩家累计赚取达到阈值时，可触发声望重置：

| 声望等级 | 累计赚取要求 | 声望点奖励 | 全局倍率 |
|----------|--------------|------------|----------|
| 0 | - | 0 | 1.00x |
| 1 | 10M | 10 | 1.25x |
| 2 | 100M | 25 | 1.50x |
| 3 | 1B | 50 | 2.00x |
| 4 | 10B | 100 | 2.75x |
| 5 | 100B | 200 | 3.75x |

### 5.2 重置规则
- **保留**: 声望点、声望倍率、已解锁成就、已购买的永久升级
- **清除**: 当前金币、已购房产数量、房产等级
- **解锁**: 每次声望重置解锁新内容（新物业类型、新小游戏等）

### 5.3 声望商店

声望点可购买的永久升级：

| 升级项目 | 声望点消耗 | 效果 | 最大等级 |
|----------|------------|------|----------|
| 租金加成 | 5 | 全局租金 +5% | 20 |
| 离线效率 | 10 | 离线收益从 50% → 100% | 1 |
| 小游戏加成 | 8 | 小游戏奖励 +10% | 10 |
| 价格折扣 | 15 | 房产价格 -3% | 5 |
| 自动收租 | 25 | 离线时自动收集随机事件奖励 | 1 |
| 双倍声望 | 50 | 声望点获取 ×2 | 1 |

## 六、离线收益系统

### 6.1 离线计算公式
```
离线时间 = min(当前时间 - 上次保存时间, 最大离线时间)
离线收益 = 每秒总收入 × 离线时间 × 离线效率
```

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 最大离线时间 | 8 小时 | 防止过度积累 |
| 离线效率 | 50% | 离线时收入减半 |
| 离线效率（升级后） | 100% | 声望商店购买后 |

### 6.2 离线结算界面
玩家重新打开游戏时显示：

## 七、随机事件详细设计

### 7.1 催租事件

```
触发条件: 每 2-5 分钟随机触发
持续时间: 8 秒
操作: 点击"催租"按钮
机制: 每次点击增加进度条，进度满时获得奖励

进度条: ████████░░ (80%)
奖励: 10-30 秒租金（根据完成度）

租客台词随机:
- "房东，再宽限几天吧..."
- "工资还没发，下个月一定交！"
- "我这就转给你！"
```

### 7.2 维修事件

```
触发条件: 每 3-8 分钟随机触发
持续时间: 5 秒
操作: 看到"维修"按钮亮起时点击
机制: 按钮随机闪烁，在亮起瞬间点击

成功: 获得 5-15 秒租金
失败: 获得 2 秒租金（保底）

维修项目随机:
- 🔧 水管爆裂
- 💡 电路故障
- 🚪 门锁损坏
- 🪟 玻璃破碎
```

### 10.3 核心系统设计（实际实现）

#### A. 游戏状态 (game_state.gd)
- **货币管理**: `money`（当前金币）、`pending_rent`（待领取租金）、`total_earned`（累计赚取）
- **房产数据**: `properties` Dictionary → `{property_id: {count: int, level: int}}`
- **家具收藏**: `furniture` Dictionary → `{furniture_id: count}`
- **声望数据**: `prestige_level`, `prestige_points`
- **升级**: `upgrades` Dictionary → `{upgrade_id: level}`
- **信号系统**: `money_changed`, `pending_rent_changed`, `income_per_second_changed`, `prestige_changed` 等
- **关键方法**:
  - `add_gold(amount)`: 增加金币并触发信号
  - `add_pending_rent(amount)`: 增加待领取租金
  - `collect_pending_rent()`: 领取全部待领取租金
  - `get_furniture_bonus()`: 计算家具总租金加成
  - `get_global_multiplier()`: 综合升级和家具的全局倍率
  - `get_prestige_multiplier()`: 声望倍率
  - `reset_for_prestige()`: 声望重置（保留家具收藏）

#### B. 经济计算 (economy.gd)
- **房产数据**: `PROPERTY_DATA` 常量字典，包含 7 种房产的价格、租金、增长率
- **被动收入定时器**: 每 0.1 秒调用 `_on_income_tick()`，将收入加入 `pending_rent`
- **购买逻辑**: `buy_property()` - 花费金币，增加 count，保留当前 level
- **升级逻辑**: `upgrade_property()` - 花费金币，增加 level，level 不能超过 count
- **价格公式**: `base_cost × 1.15^count × (1 - discount)`
- **租金公式**: `base_rent × 1.08^level × global_mult × prestige_mult`

#### C. 存档系统 (save_system.gd)
- **存储格式**: JSON 文件 `user://save_data.json`
- **自动保存**: 每 30 秒自动保存
- **离线计算**: 使用 `Economy.get_income_per_second()` 计算
- **存档内容**: money, pending_rent, total_earned, prestige_level, prestige_points, properties, upgrades, furniture, achievements, last_save_time

#### D. 数值格式化 (number_formatter.gd)
- **后缀**: `["", "K", "M", "B", "T", "Qa", "Qi", "Sx", "Sp", "Oc", "No", "Dc"]`
- **方法**: `format_number()`, `format_money()`, `format_income()`

#### E. 开始界面 (start_screen.gd)
- **纯代码构建 UI**，不依赖场景文件
- **存档检测**: 自动检测存档存在性并显示存档信息
- **新游戏**: 重置所有状态到初始值（500 金币），删除旧存档
- **继续游戏**: 加载存档后切换场景

### 10.3 存档策略

|----------|------|
| 购买/升级房产 | 立即保存 |
| 声望重置 | 立即保存 |
| 小游戏完成 | 立即保存 |
| 定时保存 | 每 30 秒自动保存 |
| 游戏退出 | 立即保存 |
| 应用切后台 | 立即保存 |
